1. MỤC TIÊU KINH DOANH
-----------------------
- Phân loại tín dụng: Good Credit (1) / Bad Credit (2)
- Ưu tiên interpretability: Mô hình cần giải thích được quyết định
  cho khách hàng và kiểm toán nội bộ
- Dataset: German Credit Data (Prof. Hans Hofmann - Univ. Hamburg)

2. THONG TIN DATASET
---------------------
- Số mẫu       : 1,000 instances
- Số features  : 20 (7 numerical + 13 categorical)
- Nhãn (label) : 1 = Good Credit, 2 = Bad Credit
- Phân bố lớp  : 700 Good (70%) / 300 Bad (30%) --> Mat can bang!
- File gốc     : german.data      (categorical dạng A11, A12, ...)
- File numeric : german.data-numeric (đã mã hóa sang dạng số)

4. COST MATRIX (DOMAIN KNOWLEDGE)
-----------------------------------
             | Predict Good | Predict Bad
-------------|--------------|-------------
Actual Good  |      0       |      1
Actual Bad   |      5       |      0

=> Phân loại Bad thành Good (FN) tệ hơn 5 lần so với FP
=> Phải dùng cost-sensitive learning hoặc điều chỉnh threshold

5. TIỀN XỬ LÝ DỮ LIỆU
-----------------------
Bước 1 - Kiểm tra missing values:
  - Dataset này không có missing values chính thức
  - Một số giá trị “unknown/no account” → giữ nguyên như 1 category

Bước 2 - Mã hóa categorical (One-Hot Encoding):
  - 13 features categorical → ~45 cột nhị phân sau encoding
  - Dùng pandas.get_dummies() hoặc sklearn.OneHotEncoder
  - drop_first=True để tránh đa cộng tuyến

Bước 3 - Chuẩn hóa numeric (StandardScaler):
  - 7 features: Duration, Amount, Installment Rate, Residence, Age, Credits, Liable
  - Công thức: z = (x - mean) / std
  - Fit trên tập train, transform cả train và test

Buoc 4 - Xử lý mất cân bằng (Class Imbalance):
  - Phương pháp 1: class_weight='balanced' trong sklearn
  - Phương pháp 2: SMOTE (Synthetic Minority Over-sampling)
  - Phương pháp 3: Điều chỉnh threshold thay vì 0.5 mặc định

Buoc 5 - Chia tập dữ liệu:
  - Train: 80% (800 mẫu) | Test: 20% (200 mẫu)
  - Dùng stratify=y Để giữ tỷ lệ 70/30 ở cả 2 tập
  - random_state=42 để tái tạo kết quả

7. KẾT LUẬN & KHUYẾN NGHỊ
---------------------------
- Dataset mất cân bằng nhẹ (70/30) → nên dùng class_weight='balanced'
- Duration và Credit Amount là 2 biến numerical quan trọng nhất
- Checking Account Status là predictor mạnh nhất (categorical)
- Nên dùng threshold ≈ 0.38 thay vì 0.5 (do cost matrix 5:1)
- Mô hình ưu tiên: Logistic Regression (interpretable + performance tot)
